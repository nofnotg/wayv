# 10. Metrics & Rollout

## 1. 핵심 질문
- 사용자들은 실제로 “나만 그런 게 아니구나”를 느끼는가?
- 숫자 없는 파도 UX가 재방문을 만드는가?
- 파도 알림이 부담이 아니라 도달감으로 작동하는가?
- 휴식 모드가 이탈이 아니라 신뢰로 이어지는가?

## 2. North Star Metrics
- 첫 주 내 첫 글 작성률
- 첫 주 내 “나도 그랬어요” 반응 경험률
- 첫 주 내 “나에게 닿은 파도” 클릭률
- 4주차 재방문율
- 자발적 재작성/보관 사용률
- 휴식 모드 사용 후 복귀율

## 3. 정성 지표
온보딩 후/1주 후 짧은 질문:
- 나만 그런 게 아니라는 느낌이 들었나요?
- 말해보길 잘했다는 느낌이 있었나요?
- 이곳이 비교보다 공명에 가깝게 느껴졌나요?
- 알림이 부담스럽지 않았나요?

## 4. 금지 지표
- 단순 체류시간 극대화
- 반응 총량만으로 성공 판단
- 자극적 글 증가를 engagement로 오판

## 5. 실험 주제
### A/B 1
- 개인화 알림 문구 A/B
- “닮은 파도가 닿았어요” vs “다시 일렁이는 흐름이 있어요”

### A/B 2
- 온보딩 6문항 vs 8문항
- 완성률과 추천 만족도 비교

### A/B 3
- 홈 구조 순서
- 나에게 닿은 파도 상단 vs 오늘의 파도 날씨 상단

### A/B 4
- 반응 버튼 4종 vs 3종

## 6. 출시 단계
### Phase 0
- 내부 테스트
- seed content
- 관리자 툴 검증

### Phase 1
- 소규모 클로즈드 베타 (30~100명)
- 정성 인터뷰 집중
- 파도 로직 threshold 튜닝

### Phase 2
- 초대 기반 오픈 베타
- 알림/휴식 모드 안정화
- 아카이브/다시 일렁임 정교화

### Phase 3
- 공개 런치
- 유료 기능 실험 전 사전 검증
