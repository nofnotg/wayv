# 04. Onboarding Question Engine

## 1. 목적
온보딩은 단순 가입 절차가 아니다.  
사용자가 처음부터 “나에게 닿는 파도”를 느끼기 위한 초기 개인화 seed를 만든다.

## 2. 설계 원칙
1. 너무 길지 않다
2. 심문처럼 느껴지지 않는다
3. 답을 바꿀 수 있다
4. 모든 사람에게 같은 질문을 던지지 않는다
5. 곧바로 추천/알림 품질 향상에 연결된다
6. 사용자는 언제든 이 질문 레이어를 건너뛰거나 나중에 다시 답할 수 있다

## 3. 질문 엔진 구조
### 단계
- Step 1. 가벼운 진입 질문
- Step 2. 관심 주제 추정
- Step 3. 감정 결 추정
- Step 4. 공개 성향 추정
- Step 5. 자극 민감도 추정
- Step 6. 휴식/거리두기 선호 추정

## 4. 질문 방식
### 기본 원칙
- 하나의 질문은 한 가지 판단만
- 2~4개 선택지
- 답변이 애매해도 괜찮은 문구 사용
- 문답 자체가 위축감을 만들지 않도록 설계

### 예시 오프닝
- 요즘 어떤 흐름이 가장 자주 떠오르나요?
- 비슷한 이야기를 본다면 어떤 느낌이 먼저 들면 좋겠나요?
- 지금은 어떤 파도를 더 보고 싶나요?
- 잠시 쉬고 싶을 때는 어떤 방식이 편한가요?

## 5. 질문 풀 초안

### 5.1 주제 관련
- 요즘 가장 자주 떠오르는 흐름은 어떤 쪽인가요?
  - 일/커리어
  - 돈/생활
  - 관계
  - 가족
  - 학업/진로
  - 건강
  - 잘 모르겠어요

- 최근 가장 많이 마음에 남는 장면은 어떤 쪽에 가까운가요?
  - 말하지 못한 일
  - 반복되는 실수
  - 끝난 관계/기회
  - 아직 정리되지 않은 선택
  - 기대만큼 풀리지 않은 노력

### 5.2 감정 결 관련
- 비슷한 이야기를 봤을 때 어떤 감정이 먼저 줄어들면 좋겠나요?
  - 고립감
  - 자책감
  - 무능감
  - 불안감
  - 답답함
  - 잘 모르겠어요

- 지금 더 닿았으면 하는 분위기는 어떤 쪽인가요?
  - 조용한 공감
  - 나도 그랬다는 연결감
  - 조금 가벼운 숨통
  - 길게 머무는 위로
  - 잠시 거리두기

### 5.3 공개 성향
- 처음에는 어떤 방식이 편한가요?
  - 완전 익명으로 둘러볼래요
  - 반익명으로 가볍게 시작할래요
  - 닉네임으로 써도 괜찮아요

### 5.4 자극 민감도
- 요즘은 어떤 흐름이 덜 부담스러울까요?
  - 짧고 조용한 이야기
  - 비슷한 경험이 모이는 이야기
  - 긴 글도 괜찮아요
  - 지금은 너무 많이 보고 싶지 않아요

### 5.5 휴식/차단
- 가끔 파도에서 벗어나 쉬는 기능이 있으면 좋을까요?
  - 자주 쓸 것 같아요
  - 가끔 필요할 것 같아요
  - 아직은 모르겠어요

## 6. 분기형 로직
### 예시
- 관계 주제를 선택하면: 관계/거리감/오해/이별 관련 세부 질문
- 돈/생활을 선택하면: 생활 압박/실수/지출/부채 부담 관련 질문
- “지금은 너무 많이 보고 싶지 않아요”를 선택하면: 초기 추천량 축소 + 휴식 모드 안내

## 7. 결과 프로필 구조
```ts
type OnboardingSeedProfile = {
  topicWeights: Record<string, number>;
  emotionWeights: Record<string, number>;
  preferredWaveTone: 'quiet' | 'resonant' | 'light' | 'mixed';
  exposureTolerance: 'low' | 'medium' | 'high';
  privacyPreference: 'anonymous' | 'semi' | 'nickname';
  restModeAffinity: 'low' | 'medium' | 'high';
}
```

## 8. 초기 추천 연결
- 가장 높은 topicWeights 기준으로 3개 카테고리 우선 추천
- preferredWaveTone이 quiet면 “조용히 이어지는 파도” 비중 증가
- exposureTolerance가 low면 긴 글/거센 흐름 노출 감소
- restModeAffinity가 high면 해변 모드 안내를 빠르게 노출

## 9. 주의점
- 초반부터 너무 많은 질문 금지
- 민감한 질문은 단계적으로
- 답하지 않아도 서비스 핵심 기능 사용 가능해야 함
- 온보딩 답변은 고정값이 아니다. 행동 데이터가 쌓이면 자동 보정한다

## 10. MVP 권장 범위
- 6문항 고정 + 2문항 분기
- 답변 수정 기능
- 결과 설명 화면 없음(과도한 진단 느낌 방지)
- 추천 개선용 내부 신호로만 사용

## 11. 향후 확장
- 계절성 질문
- 재방문 시 마이크로 체크인
- 파도 민감도 슬라이더
- 휴식 모드 추천 자동화
