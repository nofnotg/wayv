## 3. 주요 사용자 흐름

### 3.1 신규 가입 플로우
1. 로그인/회원가입
2. 공개 범위 기본 설정
3. 질문형 온보딩
4. 초기 추천 파도 3~5개 노출
5. 첫 글 쓰기 또는 둘러보기 선택

### 3.2 파도 쓰기 플로우
1. 글쓰기 진입
2. 카테고리/공개범위 선택
3. 본문 작성
4. 선택적 감정 태그
5. 게시
6. 게시 완료 상태 메시지
7. 추후 유사 파도가 생기면 알려준다는 안내

### 3.3 나에게 닿는 파도 플로우
1. 홈에서 카드 진입
2. 글 미리보기
3. 상세 진입
4. 반응 / 비슷한 파동 보기 / 보관 / 뮤트
5. 관련 흐름이 계속되면 묶음 알림

### 3.4 휴식 모드 플로우
1. 홈 또는 설정에서 “해변에서 쉬기” 선택
2. 기간 선택
   - 오늘만
   - 3일
   - 7일
   - 직접 설정
3. 적용 후 추천/푸시 강도 감소
4. 복귀 메시지 없이 부드럽게 해제 가능

## 4. 화면별 구성

### 4.1 홈
#### UI 원칙
- 숫자보다 문장
- 카드마다 상태 언어
- 과도한 블루/바다 클리셰 금지
- 시각적 출렁임은 제한적으로 사용

#### 섹션 예시
- 오늘의 파도 날씨: “오늘은 관계 주제의 파도가 길게 이어지고 있어요”
- 나에게 닿은 파도: “예전에 남긴 이야기와 닮은 파도가 닿았어요”
- 조용히 이어지는 파도: “조용하지만 오래 머무는 흐름이에요”
- 다시 일렁이는 파도: “한동안 잠잠했던 주제가 다시 움직이고 있어요”

### 4.2 글쓰기
필드:
- 제목(옵션)
- 본문
- 카테고리
- 감정 태그(옵션)
- 공개 범위
- 보관 옵션

보조 문구:
- “정리되지 않아도 괜찮아요”
- “한 문장이어도 충분해요”

### 4.3 상세
- 파도 카드 헤더
- 파도 상태 문장
- 본문
- 반응 영역
- 비슷한 파동
- 작가가 공개한 프로필 메타
- 신고/뮤트/차단

### 4.4 알림
카테고리:
- 나에게 닿은 파도
- 다시 일렁이는 파도
- 조용히 이어지는 파도 업데이트
- 시스템/운영 안내

### 4.5 마이
- 내 파도 기록
- 보관함
- 공개 범위 설정
- 휴식 모드
- 알림 강도 설정
- 답변 수정(온보딩 프로필 재학습)

## 5. 카드 설계 원칙
- 제목이 없어도 카드가 성립해야 한다
- 숫자 노출 금지
- “누가 몇 명 눌렀는지” 대신 “어떤 흐름인지” 표시
- 같은 주제를 계속 노출할 때도 중복 피로감이 없도록 요약 문구 다양화

## 6. 휴식 모드 UX
### 진입 카피
- 오늘은 파도에서 벗어나 해변에서 쉴게요
- 잠시 고요를 선택해도 괜찮아요

### 적용 효과
- 푸시 최소화
- 홈 피드 개인화 섹션 축소
- 강한 재점화 카드 숨김
- “해변에서 쉬는 중” 배지 노출(본인만)

## 7. 알림 UX 원칙
- 강한 실시간성보다 부드러운 도달감
- 같은 주제 알림은 묶음 처리
- 사용자가 가장 자주 반응하는 시간대 우선
- 야간 푸시는 기본 제한

## 8. 접근성
- 색만으로 상태를 구분하지 않는다
- 텍스트 레이블 필수
- 진동/사운드 피드백은 선택 옵션
- 민감한 문구는 미리보기에서 일부 축약 가능
