# 09. Moderation, Safety & Ops

## 1. 기본 입장
wayv는 따뜻한 언어를 사용하지만, 안전과 운영 판단은 흐리지 않는다.

## 2. 운영 목표
- 독성 억제
- 실패 경쟁 프레임 차단
- 고위험 표현 대응
- 사용자 프라이버시 보호
- 커뮤니티 문화 보존

## 3. 모더레이션 범주
### 허용
- 실패 경험 공유
- 자조적 유머(타인 공격 없고 자기비하 소비 구조로 흐르지 않는 범위)
- 정리되지 않은 감정 표현

### 조정 대상
- 타인 비난/조롱
- 혐오/낙인
- 폭력적 자극
- 자해/자살 고위험 신호
- 개인정보 노출
- 반복적 스팸/낚시

## 4. 위험도 레벨
### Low
- 날카로운 표현
- 커뮤니티 톤과 어긋나는 말
- 부드러운 수정 유도

### Medium
- 반복 공격성
- 과도한 비방
- 노출 제한 + 수동 리뷰

### High
- 자해/자살 구체 표현
- 실명/전화번호/주소 노출
- 불법 행위 조장
- 즉시 제한 + 관리자 검토

## 5. 게시/댓글 조정 흐름
1. 자동 탐지
2. soft warning / rewrite prompt
3. user resubmit
4. needed 시 limited 상태
5. admin review
6. audit log 저장

## 6. 위기 대응
### Tier 1
- 경미한 절망 표현
- 안전 자원 노출 + 부드러운 안부 문구

### Tier 2
- 반복적 자해 언급, 구체 계획 없음
- 위기 리소스 강화, 빠른 도움 CTA
- 관리자 큐 진입

### Tier 3
- 즉각적 위험으로 보이는 표현
- 최우선 관리자 검토
- 플랫폼 내 노출 제한
- 지역 기반 긴급 도움 정보 우선 노출

## 7. 운영 문구 예시
- 이 이야기가 안전하게 머물 수 있도록 표현을 조금 함께 다듬어볼게요
- 지금은 빠르게 닿을 수 있는 도움을 먼저 열어둘게요

## 8. 실패 경쟁 방지 규칙
- 수치 랭킹 금지
- 자극적 텍스트 증폭 금지
- “오늘의 파도”는 글 단위가 아닌 분위기 단위
- 자극성 score를 파도 증폭 로직에서 감산 요인으로 둘 수 있음

## 9. 운영 대시보드 요구사항
- open cases
- recent reports
- high severity queue
- repeated offender history
- post state timeline
- action log

## 10. 관리자 SLA (권장)
- High: 15분 ~ 1시간 이내
- Medium: 24시간 이내
- Low: 72시간 이내

## 11. 정책 문서 별도 필요
- 커뮤니티 가이드라인
- 개인정보 처리방침
- 신고 정책
- 휴식 모드/알림 정책
- 고위험 표현 대응 안내
