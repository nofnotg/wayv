# 01. PRD

## 1. 제품명
wayv

## 2. 제품 카테고리
실패 경험 기반 경험 공유 플랫폼 / 사회적 외부화 플랫폼

## 3. 문제 정의
많은 사람은 실패를 겪고도 밖으로 말하지 못한다.  
그 결과 고립감, 외로움, 자책감, 무능감이 자기 정체성처럼 굳어진다.

기존 플랫폼은 다음 한계를 가진다.
- 일반 SNS: 성공 전시장 성격이 강함
- 익명 커뮤니티: 독성/가벼운 소비로 흐르기 쉬움
- 상담/멘토링 서비스: 조언 중심, 진입 장벽 높음
- 공공 지원 서비스: 공감 구조보다 정보 제공에 치중

wayv는 실패를 해결책으로 밀어붙이지 않고, **말해도 되는 경험**으로 바꾸는 구조를 제공한다.

## 4. 타깃 사용자
### 핵심 타깃 (MVP)
- 20~40대
- 실패 경험을 말하고 싶지만 공개 SNS에는 올리기 어려운 사람
- 조언보다 공감과 외부화를 먼저 필요로 하는 사람
- 익명 또는 반익명 표현을 선호하는 사람

### 확장 타깃
- 전 연령
- 커리어, 관계, 학업, 돈, 일상, 가족 등 다양한 실패 경험을 가진 사람

## 5. MVP 목표
### 목표
- “실패를 외부화했을 때 고립감이 줄어드는가”를 검증
- 파도 UI/알림이 숫자 없는 공명 경험을 만들 수 있는지 검증
- 개인화된 “나에게 닿는 파도”가 재방문 동기가 되는지 검증

### 비목표
- 전문가 상담 플랫폼 완성
- 수익화 전면 적용
- 고도화된 AI 멘토 기능
- 대규모 B2B 기능

## 6. MVP 핵심 기능

### 6.1 회원가입/인증
- 이메일 또는 소셜 로그인
- 기본 익명 닉네임 발급
- 공개 프로필 수준 선택
  - 완전 익명
  - 반익명(연령대/관심태그/상황태그 공개)
  - 닉네임 공개
- 푸시 알림 동의 선택
- “신뢰 연락처”는 후순위 옵션으로 설계만 반영

### 6.2 질문형 온보딩
- 첫 가입 시 5~9개의 짧은 질문
- 사용자의 최근 실패 결, 관심 주제, 공감하고 싶은 분위기 파악
- 모든 사용자에게 동일 질문이 아니라 분기형 질문 세트 적용
- 온보딩 결과는 초기 “나에게 닿는 파도” 추천 seed로 사용
- 사용자는 답변 수정 가능해야 함

### 6.3 파도 쓰기
- 제목(선택), 본문(텍스트)
- 카테고리 선택
- 감정 태그 선택(선택)
- 공개 범위 선택
- “이 파도를 나만 보관할래요” 옵션
- 작성 완료 후 상태 메시지 표시

### 6.4 파도 보기
메인 홈 섹션:
- 오늘의 파도 날씨
- 나에게 닿은 파도
- 조용히 이어지는 파도
- 다시 일렁이는 파도

### 6.5 반응 버튼
초기 반응 후보:
- 닿았어요
- 나도 그랬어요
- 파도를 보탤게요
- 조용히 머물게요

반응은 내부 가중치가 다르며, 공개 수치는 보여주지 않는다.

### 6.6 알림
- 나에게 닿는 파도 알림
- 예전에 쓴 글과 닮은 파도 알림
- 예전에 공감했던 흐름이 다시 일렁일 때 알림
- 알림은 빈도 제한과 묶음(digest) 규칙 적용

### 6.7 휴식 모드
사용자가 외부 자극을 줄이고 싶을 때:
- “오늘은 파도에서 벗어나 해변에서 쉴게요”
- 일정 시간 또는 일정 기간 동안 추천/푸시/홈 강한 노출 완화
- 복귀는 언제든 가능

### 6.8 파도 수명 주기
- 시간이 지나면 기본 노출이 자연스럽게 약해진다
- 오래된 파도는 아카이브로 이동
- 사용자는 일부 파도를 유지/보관할 수 있다

## 7. 기능 상세 요구사항

### 7.1 홈 화면
#### 오늘의 파도 날씨
- 개별 글 인기 노출 금지
- 카테고리 또는 전체 흐름의 상태만 표현
- 예: “오늘은 일과 관계 주제의 파도가 길게 이어지고 있어요”

#### 나에게 닿은 파도
- 개인화 추천 최상단
- 사용자가 쓴 글에 높은 가중치
- 비슷한 주제, 감정, 최근 반응 흐름 반영

#### 조용히 이어지는 파도
- 극단적 급등이 아닌, 꾸준히 공명하는 글
- 숫자 대신 상태 문장 사용

#### 다시 일렁이는 파도
- 한동안 잔잔했다가 다시 공명이 생긴 글
- 회상, 계절성, 주제 재점화 흐름 반영

### 7.2 상세 화면
- 글 본문
- 작성자 공개 수준에 따른 정보
- 파도 상태 문장
- 반응 버튼
- 비슷한 파동 보기
- 아카이브/보관/뮤트

### 7.3 알림 정책
- 실시간보다 “잘 닿는 타이밍” 우선
- 동일 사용자에게 하루 최대 n회
- 낮/밤 시간대 차등
- 과다 자극 방지 위해 묶음 알림 우선
- 휴식 모드 중엔 푸시 중지 또는 최소화

## 8. 사용자 스토리
- 나는 처음 가입했을 때 내 이야기를 길게 쓰기 전에, 몇 개의 질문으로 나와 닮은 파도를 먼저 만나고 싶다.
- 나는 내 글에 숫자가 붙기보다, 누군가에게 닿았다는 감각을 받고 싶다.
- 나는 다른 사람의 실패를 소비하지 않고, 함께 살아가는 느낌을 받고 싶다.
- 나는 피곤할 때 잠시 파도에서 벗어나 쉬고 싶다.
- 나는 내 과거 글과 닮은 흐름이 다시 생기면 조용히 알려받고 싶다.
